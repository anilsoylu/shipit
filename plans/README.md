This repo is a **docs/template pack** (markdown only — no build, test, or lint).
"Verification" in every plan is file-existence (`ls`), content-presence
(`grep`), URL/slug liveness (`curl`), and cross-reference consistency, plus human
review of rendered markdown. No compiler catches mistakes — the grep/curl gates
are the safety net.

## Goal

Make the template **pluggable and extensible to any stack** without weakening
its decisive defaults, and make it **comfortable for vibe-coding** (decide the
stack in seconds, every choice auto-loads its skill + `llms.txt`). Architecture:
per domain, a **contract** (durable invariants) + an **add-your-own template**
(extensible to any tool) + **reference playbooks** (turnkey for common stacks),
all indexed by a single verified **LLM Docs Registry** and fronted by a fast
**stack-picker**.

## Known tradeoff (accepted by the user)

The full contract+template+playbook approach maximizes optionality and comfort
but adds ~7 router docs + many reference playbooks with no CI. The contract +
template layers hold the durable rules and change rarely; version-specific detail
lives in playbooks and the registry (which carry a verified-on date). Reviewers
should treat version-specific claims (skill slugs, `llms.txt` URLs, SDK APIs) as
the highest-risk lines and re-verify with `curl` — every plan makes that an
explicit gate with a STOP-and-report escape hatch on 404s. Suspect aggregator
skill slugs surfaced by research (e.g. `agents-inc/skills`) are flagged in
`research-findings.md`; prefer the vendor's own skill or drop it.
